# 현금 보상 레벨 기준 — 근거 문서

> 최종 레벨 현금 보상이 "왜 Lv.4 / 20,000원인가"를 숫자로 설명하는 문서.
> 모든 수치는 코드에서 직접 계산했고 출처를 함께 적었다.
>
> **출처**
> - 레벨 곡선: `server/app/services/points.py` — `LEVEL_BASE = 50`, `LEVEL_INC = 150`
> - 적립 규칙: `server/app/services/points.py` — `POINT_RULES`
> - 보상 기준: `server/app/core/config.py` — `final_level = 4`, `final_level_reward_amount = 20000`
> - 핵심 규칙: 레벨은 **XP로만** 오르고, XP는 적립 포인트와 **같이** 쌓인다(같은 행동 = 같은 양). CP(소비 포인트)와는 무관.

---

## 0. 한눈 요약 (TL;DR)

| 항목 | 값 |
|---|---|
| 현금 받는 기준 | **Lv.4 도달** |
| Lv.4에 필요한 누적 XP | **600 XP** |
| 보상 금액 | **20,000원** (사용자당 1회) |
| 보통 사용자 기준 도달 시간 | **약 2주** |
| 받는 방법 | 포인트 탭 → "받기" 신청 → 관리자 확인 후 지급 |

핵심 메시지: **"2주 동안 꾸준히 기록한 사람"에게만 현금이 가도록** 설계된 기준이다.

---

## 1. 레벨 곡선 — 레벨마다 필요한 XP

레벨은 한 칸 오를 때마다 비용이 무거워진다(등차수열).

```
Lv.N → Lv.N+1 에 드는 XP = 50 + (N-1) × 150
```

| 구간 | 그 칸에 드는 XP | **도달까지 누적 XP** |
|---|---|---|
| Lv.1 (시작) | — | 0 |
| Lv.1 → 2 | 50 | 50 |
| Lv.2 → 3 | 200 | 250 |
| **Lv.3 → 4** | **350** | **600** ← 현금 기준 |
| Lv.4 → 5 | 500 | 1,100 |
| Lv.5 → 6 | 650 | 1,750 |
| Lv.6 → 7 | 800 | 2,550 |
| Lv.7 → 8 | 950 | 3,500 |

> 누적 XP 공식: `n × 50 + 150 × n × (n-1) / 2` (n = 레벨-1).
> 예) Lv.4 → n=3 → `3×50 + 150×3×2/2 = 150 + 450 = 600`. ✔

---

## 2. XP를 주는 행동 6가지

`POINT_RULES` 그대로. 포인트와 XP가 동시에 같은 양 적립된다.

| 행동 | 주는 XP | 빈도 |
|---|---|---|
| 식단 1회 기록 | **10** | 끼니마다 (하루 최대 3끼 인정) |
| 하루 3끼 완료 | **20** | 하루 1회 (아침·점심·저녁 다 기록 시) |
| 주 5일 기록 | **100** | 주 1회 (그 주에 5일 기록 달성 시) |
| 운동 1회 기록 | **10** | 운동할 때마다 |
| 주 3일 운동 보너스 | **80** | 주 1회 (서로 다른 3일 운동 시) |
| 설문 완료 | **50** | 1회성 |

**행동 강도별 하루/주 적립량**

- 하루 풀로 기록(3끼 + 운동): `30 + 20 + 10 = 60 XP / 일`
- 주간 보너스 다 받으면: `100 + 80 = 180 XP / 주`
- → 매일 풀로 쓰는 사람의 **이론상 최대 ≈ 600 XP / 주** (+ 첫 주 설문 50)

즉 **가장 열심히 쓰면 1주 만에도** Lv.4(600)가 가능하고, **보통은 2주** 걸린다.

---

## 3. 사용자 유형별 — Lv.4까지 걸리는 시간

가정을 명시한 3가지 시나리오. (주간 보너스는 그 주 조건을 채운 시점에 들어온다)

### A. 매우 성실 — 매일 3끼 + 운동, 주간 보너스 다 챙김
- 주당 적립: `60×7 + 180 = 600 XP` (+ 첫 주 설문 50 = 650)
- **W1 종료 시점에 이미 600 도달 → 약 1주**

### B. 보통 — 3끼는 챙기고(주 5일), 운동은 주 2일, 설문 1회
- 주당 적립: `(30+20)×5 + 100(주5일) + 20(운동2회) ≈ 370 XP` (+ 첫 주 설문 50)

| 주차 | 그 주 적립 | 누적 XP | 레벨 |
|---|---|---|---|
| W1 | 420 | 420 | Lv.3 |
| **W2** | 370 | **790** | **Lv.4 도달** ✅ |
| W3 | 370 | 1,160 | Lv.5 |

→ **약 2주** (현재 기준이 겨냥하는 표준 사용자)

### C. 라이트 — 하루 1~2끼만, 주 3~4일, 운동·주간보너스 거의 못 챙김
- 주당 적립: `약 60 XP` (주 5일 미달이라 100점 보너스 없음)
- 600까지: `(600-50) / 60 ≈ 9주`
- → **약 2달 이상** (꾸준하지 않으면 사실상 도달이 느림)

> 의도된 결과: **대충 쓰는 사람에게는 현금이 잘 안 간다.** 보상이 자연스럽게 "정착한 사용자"에게만 흘러간다.

---

## 4. 왜 하필 Lv.4(600 XP / ≈ 2주)인가 — 근거

1. **습관 형성의 첫 분기점이 2주.**
   행동과학에서 단순·반복 행동이 루틴으로 자리 잡기 시작하는 최소 구간이 약 2~3주다(흔히 인용되는 "2주/21일" 통념). **2주 연속 기록**은 "한번 써봤다"와 "습관이 들기 시작했다"를 가르는 선이고, 보상을 여기에 거는 것이 리텐션 관점에서 가장 효율적이다.

2. **600 XP = 보통 사용자의 딱 2주치 적립량.**
   3절 시나리오 B처럼, 우리 적립 규칙으로 평범하게 쓰면 2주에 600 언저리가 쌓인다. 즉 "2주 꾸준히"라는 목표와 XP 수치가 **계산상 정확히 맞물린다** — 임의로 정한 숫자가 아니다.

3. **너무 낮지도, 너무 높지도 않다.**
   - Lv.2(50 XP)는 **하루**면 도달 → 어뷰징·동기 희석, 현금 줄 의미 없음.
   - Lv.8(3,500 XP)은 매일 풀로 써도 **6주+**, 보통은 2~3달 → 대부분 도달 전에 이탈.
   - Lv.4는 "진심으로 2주 써본 사람"만 통과하는 **최소한의 진입 장벽**이다.

4. **금액(20,000원)의 의미.**
   2주 노력의 대가로 체감되는 금액이면서, 1회성이라 사용자당 비용이 고정된다(반복 지급 아님). 동기부여는 충분하되 비용은 예측 가능.

**결론:** Lv.4는 "감"이 아니라 *(2주 습관 형성) × (실제 적립 곡선)* 두 근거가 만나는 지점이다.

## 5. 기준을 바꾸고 싶다면 (대안 비교)

목표가 "더 오래 정착한 사람만"이라면 기준 레벨을 올리면 된다. 시간은 **보통 사용자(주 ≈ 370 XP)** 기준 추정.

| 기준 레벨 | 누적 XP | 보통 사용자 도달 시간 | 성격 |
|---|---|---|---|
| Lv.3 | 250 | 약 1주 | 너무 쉬움 (어뷰징 위험) |
| **Lv.4 (현재)** | **600** | **약 2주** | **습관 형성 진입 — 권장 기본값** |
| Lv.5 | 1,100 | 약 3주 | 살짝 더 타이트하게 |
| Lv.6 | 1,750 | 약 5주 | 한 달+ 장기 정착자 대상 |
| Lv.8 | 3,500 | 약 2~3달 | 헤비 유저 전용 (도달률 낮음) |

**의사결정 가이드**
- **정해진 인원 대상 파일럿/연구** → Lv.4 그대로 추천 (2주 리텐션 목표에 최적, 비용 = 20,000 × 인원으로 예측 가능)
- **불특정 다수 공개 앱** → Lv.6~8로 올리거나 금액을 5,000~10,000원으로 낮추고, 어뷰징 가드(계좌 1회 제한·본인인증) 추가 권장

**바꾸는 법** — 코드 두 줄만 수정:
```python
# server/app/core/config.py
final_level: int = 4                  # 기준 레벨
final_level_reward_amount: int = 20000  # 보상 금액(원)
```
레벨 곡선 자체(난이도)를 바꾸려면 `server/app/services/points.py`의 `LEVEL_BASE` / `LEVEL_INC` 두 값만 조정하면 표 전체가 따라 바뀐다.
